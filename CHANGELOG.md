  만들었다 — 이 중 하나라도 빠지면 **오류 없이 아무 일도 안 일어나기** 때문이다.

## [0.13.0] - 2026-07-15

### Changed
- **설정 Export 이식성**: 설정 Export가 사용자 자판(`.jmt`) 본문을 config.ini에 함께
  동봉(`[LayoutFile:...]` 섹션)한다. 다른 PC에서 Import하면 설정뿐 아니라 사용자 자판까지
  `%APPDATA%\Jamotong\layouts`로 복원된다(없던 것만; 재시작 후 사용 가능). Apply(자동 저장)는
  동봉하지 않는다.
- **`jamotong.exe`가 트레이 상주 → 일반 관리 앱으로 승격**: 시스템 트레이 아이콘/백그라운드
  폴링을 없애고, 작업 표시줄·작업 관리자에 나오는 일반 창 앱이 됨. 메뉴 기반으로 **`.jmt`
  자판 파일 열기/편집/저장 + 로드 검증(파서 진단 = 줄 번호·사유 표시)**, 같은 에디터에서
  **입력 테스트(TSF 없이 자판/오토마타 검증)**, 설정 창 열기, 정보를 제공. 명령행에 `.jmt`
  경로를 주면 바로 열림.

### Added
- **한자 훈음 표 대폭 확장(1,784자 → 6,832자)**: 인명용 보강분 등 그동안 뜻 없이 음(音)만
  표시되던 한자 약 5,000자에 전통 대표 훈음을 추가했다(예: `珉 옥돌 민`, `埈 높을 준`).
  후보창·코드포인트 팝업에서 한자 뜻 식별이 쉬워진다. 훈음은 Unicode Unihan 영문 정의
  (Unicode License)의 의미를 근거로 새로 작성했고(사전 문구 전사 없음), 음이 변환 테이블과
  일치하는지 검증했으며, 대표 훈을 확정하기 어려운 벽자는 넣지 않았다(음 폴백 유지).
- **코드포인트 입력 팝업에 문자명 표시**: `Ctrl+Alt+U` 팝업이 현재 코드포인트의 이름을 보여줌 —
  한자는 훈음("집 가")·음("가"), 그 외는 유니코드 블록명(Hangul, Kana, Emoji 등).
- **한자 후보창이 훈음 없는 한자도 음을 표시**: 훈음 미수록 한자가 `U+XXXX` 대신 음("特 특")으로
  표시됨(한자→음 역인덱스). 새로 늘어난 인명용 한자·복합어에 특히 유용.
- **한자 단어(복합 한자어) 대폭 확장**: 단어 단위 한자 변환 후보를 331개 → 약 2,200개로 확장
  (예: 학교→學校, 사회→社會 외 상용 한자어 다수). 자체 큐레이션 331개를 각 단어의 최우선
  후보로 유지하고, `jemdiggity/hanja-wordlist`(MIT)에서 **한글↔한자 매핑만** 보강(뜻풀이는
  취하지 않음). 복합어만·어미 정리·표준 CJK 필터 적용.
- **인명용 한자 대폭 보강**: 이름에 쓰이는 표준 한자 1,060자가 그동안 변환되지 않던 문제 해결
  (원인: Unicode Unihan `kHangul`이 독음 등록 한자만 수록). 대한민국 대법원 인명용 한자 목록
  (공공데이터, 경유 배포 rutopio/Korean-Name-Hanja-Charset MIT)으로 보강해 인명용 표준 한자를
  100% 커버(고유 한자 8,465 → 9,525자). 표준 폰트에 없는 이체자(PUA·미할당)는 제외.

### Security
- **설정 Import 시 경로 조작으로 임의 위치에 파일이 생성되던 문제 차단**: Export 번들의
  `[LayoutFile:이름]` 항목 이름을 검증 없이 `%APPDATA%\Jamotong\layouts\<이름>`으로 복원해서,
  신뢰할 수 없는 config.ini를 Import하면 `..\..\`가 포함된 이름으로 자판 폴더 밖(예: 시작
  프로그램 폴더)에 임의 확장자 파일을 심을 수 있었다. 이제 경로 구분자가 없고 `.jmt` 확장자이며
  Windows 예약 장치명(CON·COM1 등)이 아닌 파일명만 복원한다. 복원 개수도 자판 목록 크기(8)로
  제한해 악의적 번들이 사용자 자판을 밀어내지 못하게 했다. 파일명에 들어갈 수 있는 `]`·`%`는
  헤더에서 percent-encoding으로 안전하게 싣고 복원 시 되돌린다. 번들 본문도 선두 공백 마커로
  감싸 내부의 `[` 로 시작하는 줄이 상위 설정 파서로 오인되지 않게 했다.

### Fixed
- **`_snwprintf` 경로 조합 시 널 종료 누락으로 인한 버퍼 과다읽기 방지**: 자판 폴더/파일 경로
  조합에서 경로 합계가 260자를 넘어 잘릴 때 널 종료가 붙지 않아 뒤이은 파일 열기가 버퍼 경계를
  넘어 읽을 수 있었다. 조합마다 명시적으로 널 종료를 보장한다.
- **PuTTY 등 자체 렌더링 터미널에서 한글 입력이 일정량 이후 씹히던 문제 방지**: EDIT 계열
  판정이 클래스명 없이 `EM_GETSEL`/`EM_EXGETSEL` 응답만 봐서, 비-EDIT 창이 우연히 응답하면
  `EM_REPLACESEL`(그 창이 무시)로 커밋해 입력이 사라질 수 있었다 — 포커스 컨트롤의 클래스명에
  `edit`가 있어야 EDIT 계열로 인정하도록 강화(PuTTY는 TSF 경로 유지).

### Changed
- **설정 창 색 팔레트 정리**: 라이트 모드에서 창 배경(`COLOR_BTNFACE`)과 컨트롤 배경
  (`COLOR_WINDOW`)을 섞어 써 생기던 미묘한 배경색 차이를 제거하고, 창·라벨을 하나의 배경색
  으로 통일(라이트 = Win11 톤, 다크 = 정제). 입력 필드(글꼴 표시 라벨·리스트·콤보)는 대비되는
  컨트롤 배경으로 구분. 버튼은 호환성을 위해 시스템 스타일 유지.
- 라벨·체크박스 텍스트 배경을 투명 처리 — 탭 컨트롤 안(살짝 밝음)과 창 위(살짝 어두움)의
  배경색이 달라 라벨 주위에 사각형 seam이 보이던 문제 해결(라벨이 아래 배경색을 그대로 따름).
- **AkelPad(AkelEdit)에서 입력·한자 교체가 반영되지 않던 근본 문제**: AkelEdit는 TSF
  `InsertTextAtSelection`을 `hr=0`(성공)으로 받고도 문서에 실제로 반영하지 않았다(진단 로그
  상 커밋해도 캐럿 좌표가 고정 — 음절이 사라지거나 특정 글자가 반복 입력 안 됨, 한자 교체가
  앞 글자만). **EDIT 계열(EM_EXGETSEL/EM_GETSEL에 응답하는 컨트롤)에서는 커밋 삽입·선택
  교체를 모두 `EM_REPLACESEL`로** 수행하도록 전환(터미널·네이티브 리치앱 등 비-EDIT는 종전
  TSF 경로 유지). 후보창 콜백 시점엔 포커스가 옮겨가 대상 창을 못 찾던 문제도, 한자키 시점의
  대상 EDIT 창을 저장해 해결.
- **한자 변환이 AkelPad에서 앞 두 글자만 교체되던 문제**("대한민국"→"大韓민국"; 메모장·카톡은
  정상): EDIT 계열 판정을 `EM_GETSEL`로 했는데 AkelEdit는 이 메시지에 응답하지 않아, 교체
  헬퍼가 실패로 떨어져 CUAS에서 부분 교체하는 TSF 삽입으로 폴백됐다 — 선택 읽기가 이미
  성공하던 **`EM_EXGETSEL`을 EDIT 판정에 우선 사용**하도록 고쳐 `EM_REPLACESEL`(선택 전체
  교체)/`EM_EXSETSEL`(단어 선택)이 AkelEdit에서도 동작한다.
- **한자 변환 도중 취소하면 조합 중이던 원본 글자가 사라지던 문제**("가" 입력 → 한자키 →
  선택 없이 나오면 "가" 소실): 조합 중 음절/자모를 후보 삽입 전에 **문서에 먼저 커밋**하고
  선택 시 그 글자를 교체하도록 바꿔, 취소해도 원본이 그대로 남는다.
- **레거시 앱에서 조합 중 엔터 시 마지막 음절 소실이 지연 재전달로도 재발**: SendInput은
  시스템 입력 큐라 앱 큐의 확정문자 전달과 순서 보장이 없다 — EDIT 계열로 판정되는 창에는
  **WM_KEYDOWN/UP을 그 창의 메시지 큐에 직접 게시**(같은 큐 FIFO = 추월 불가)하고,
  터미널 등 비-EDIT 창만 SendInput을 유지.
- 레거시(CUAS) 앱에서 **블록 선택 → 한자키 변환이 무동작**이던 문제(메모장 등 네이티브
  앱은 정상): 레거시 EDIT 계열은 선택을 TSF `GetSelection`으로 노출하지 않는다 — TSF
  읽기가 빈손이면 포커스 컨트롤에서 직접 읽는 폴백 추가. 1차(EM_GETSEL+WM_GETTEXT)는
  일부 앱에서 오프셋 불일치로 "대한민국"이 "대한"으로 잘려, **RichEdit 계열 정확 경로
  (`EM_EXGETSEL`+`EM_GETSELTEXT` — 컨트롤이 선택 텍스트를 직접 복사)를 1차로**, 플레인
  EDIT용 EM_GETSEL을 2차로 쓰는 2단 폴백으로 보강.
- **특정 글자가 계속 무시되던(입력이 씹히던) 갇힘 현상**: 모아치기/코드 자판 상태 머신이
  keyup 유실로 박힌 '유령 키'를 오토리핏으로 오인해, 그 키를 영원히 먹기만 하고 조합을
  확정하지 않았다(갇힌 조합이 미리보기 칩에 계속 표시). 반복 판정 시 OS 물리 키 상태와
  대조해 자가 치유하도록 수정.
- **조합/미리보기 탈출·정리 강화**: `Esc` = 조합 취소(확정하지 않고 비움 — MS IME 관례,
  순차·모아치기 공통), 모아치기 조합 중 백스페이스 정리 정상화(이전엔 조건이 잘못돼
  동작 안 함), 자판 전환 시 모아치기 잔여 상태 정리. 조합·칩 상태 리셋을 단일 함수
  (`ResetComposition`)로 중앙화 — 흩어진 정리 경로에서 칩이 갇히는 부류의 재발 방지.
- 미리보기 칩이 여러 글자 preedit(다국어 확장 대비)를 표시할 수 있게 버퍼 확장(최대 63자).

## [0.12.0] - 2026-07-08

### Added
- **기능별 복수 단축키**: 자판 전환뿐 아니라 **한자 변환**, **유니코드 코드 입력**(기존
  Ctrl+Alt+U 고정), **설정 창 열기**(기존 Ctrl+Alt+K 고정)도 단축키를 변경하거나 여러 개
  지정 가능(기능당 최대 8개).
- 설정 창 **Shortcuts 탭 개편**: 위 콤보에서 기능(자판 전환/한자 변환/코드 입력)을 고르고,
  아래 리스트에서 그 기능의 단축키를 추가(Add)/편집(Edit)/삭제(Del). 한자 키 설정은
  IME Options 탭에서 Shortcuts 탭으로 이동.
- **`.jmt` 배열(키 나열) 지정 문법**: 좌변에 키를 여러 개 쓰면 한 줄로 일괄 배정 —
  `Map qwertyuiop = ',.pyfgcrl`(static, 위치 대응) / `Key khj = C0 C2 C11`(hangul,
  키 수=스펙 수) / `Key jkl; = 0`(chord, 시작 비트부터 연속). 단건 지정과 완전 호환,
  개수 불일치·범위 밖은 로드 거부. 예제 3종·README 레퍼런스 갱신.

### Changed
- 설정 파일 `[Shortcuts]` 구조를 기능별로 재편(`RotateCount/Rotate0_Key…`,
  `HanjaCount/…`, `CodeCount/…`). 구형(~v0.11) 파일의 `Count/0_Key…`(자판 전환)와
  `[Options] HanjaKey/HanjaMods`(단일 한자키)는 로드 시 자동 변환(하위호환).
- 단축키 캡처 팝업이 설정 창과 동일한 글꼴(Segoe UI, DPI 스케일)을 사용.
- Apply & Save 버튼 폭을 캡션 좌우 여백(~0.1em)만큼 확장.

### Docs / Packaging
- **README 전면 개편**(영/한, 상호 링크): 설치(원하는 폴더 복사 → `install.bat`,
  예: `C:\Program Files\Jamotong`)·기본 자판/기본 키 표·설정 파일 위치
  (`%APPDATA%\Jamotong\config.ini`)·삭제 절차와 파일 잠금 설명, 그리고

### Fixed (RFC-0004 소스 리뷰 반영)
- **조합 중 앱 단축키(Ctrl/Alt/Win 조합)가 조합을 확정하지 못하던 문제**: 조합 중 Ctrl+S를
  누르면 마지막 음절이 빠진 채 저장됐다. 이제 조합 중 단축키는 **음절을 먼저 확정하고 키는
  그대로 통과**시켜 앱이 원래 타이밍에 기능키를 처리한다. (1차 구현의 eat+재전달 방식은
  실기에서 주입 키가 큐의 사용자 입력에 추월당해 Ctrl 없이 자모로 오입력되는 문제(ㅊ/ㅍ)가
  발견되어 통과 방식으로 재설계.)
- **레거시(CUAS) 앱에서 조합 중 엔터를 누르면 마지막 음절이 소실되던 문제**(AkelPad
  "가나다라"+Enter → "가나다"): 경계키(엔터/방향키 등) 재전달을 ~30ms 지연해 확정 음절이
  앱에 먼저 도달하게 했다. (스페이스는 v0.11.0의 단일 삽입 유지.)
- **조합 미리보기 칩이 매 키마다 크기가 바뀌며 여러 번 깜빡이던 문제**(메모장): 사후
  자기보정이 편집세션 rect와 시스템 캐럿 rect의 항구적 높이 차이를 서로 "교정"하며 진동
  — 보정을 캐럿 위치가 실제로 움직였을 때만 발동하도록 수정.
- **설정 창 Add로 추가한 외부 `.jmt` 자판이 재시작 후 사라지던 문제**: 추가 시
  `%APPDATA%\Jamotong\layouts`(사용자 자판 저장소)로 복사하고, 시작 시 DLL 옆 폴더와 함께
  자동 로드한다.
- 설정 창에서 마지막 켜진 자판을 삭제해 "켜진 자판 0개" 상태를 만들 수 있던 문제(적용 시
  자동 복구 포함), Add 후 적용 전 삭제한 자판의 메모리 누수.
- IME 비활성화 시 열려 있던 한자 후보창이 정리되지 않던 문제(해제된 서비스 참조 위험).
- 잘못된 `.jmt`(범위 밖 자모 인덱스, 정의 안 된 조합 글쇠, 미지의 key/mod 이름)가 "로드
  성공 후 입력만 깨지는" 대신 로드 시점에 거부된다.
- `regsvr32` 등록 시 TSF 프로파일/카테고리 등록 실패가 성공으로 숨겨지던 문제 — 실패
  코드가 설치 스크립트까지 전파된다.
- 편집 세션 내부 실패(삽입 실패 등)가 호출자에 전파되지 않던 문제.
