[Semantic Versioning](https://semver.org)(MAJOR.MINOR.PATCH — 비호환 변경 / 신규 기능 / 버그 수정)을 따른다.

## [Unreleased]

## [0.12.0] - 2026-07-08

### Added
- **기능별 복수 단축키**: 자판 전환뿐 아니라 **한자 변환**, **유니코드 코드 입력**(기존
  Ctrl+Alt+U 고정), **설정 창 열기**(기존 Ctrl+Alt+K 고정)도 단축키를 변경하거나 여러 개
  지정 가능(기능당 최대 8개).
- 설정 창 **Shortcuts 탭 개편**: 위 콤보에서 기능(자판 전환/한자 변환/코드 입력)을 고르고,
  아래 리스트에서 그 기능의 단축키를 추가(Add)/편집(Edit)/삭제(Del). 한자 키 설정은
  IME Options 탭에서 Shortcuts 탭으로 이동.

### Changed
- 설정 파일 `[Shortcuts]` 구조를 기능별로 재편(`RotateCount/Rotate0_Key…`,
  `HanjaCount/…`, `CodeCount/…`). 구형(~v0.11) 파일의 `Count/0_Key…`(자판 전환)와
  `[Options] HanjaKey/HanjaMods`(단일 한자키)는 로드 시 자동 변환(하위호환).
- 단축키 캡처 팝업이 설정 창과 동일한 글꼴(Segoe UI, DPI 스케일)을 사용.
- Apply & Save 버튼 폭을 캡션 좌우 여백(~0.1em)만큼 확장.

### Docs / Packaging
- **README 전면 개편**(영/한, 상호 링크): 설치(원하는 폴더 복사 → `install.bat`,
  예: `C:\Program Files\Jamotong`)·기본 자판/기본 키 표·설정 파일 위치
  (`%APPDATA%\Jamotong\config.ini`)·삭제 절차와 파일 잠금 설명, 그리고
  **`.jmt` 사용자 자판 문법 전체 레퍼런스**(static/hangul/chord — 레이어·원샷/토글·
  탭-홀드(0.2초 딜레이드)·지속형 홀드·마우스 동작·키 이름 목록, 예제 포함).
- 배포 zip에 `README.md`/`README.ko.md`/`LICENSE` 동봉(`README-install.txt` 대체),
  `make stage`도 동일하게 복사.
- 설치/삭제 스크립트 안내문 정비(영어 전용 유지): 설치 후 기본 자판·기본 키·설정
  파일 위치 안내, **uninstall.bat이 등록 해제 후 잠기지 않은 바이너리를 즉시 삭제**하고
  잠긴 파일은 로그아웃 후 재실행으로 정리(2단계, 재부팅 불필요).

### Fixed (RFC-0004 소스 리뷰 반영)
- **조합 중 앱 단축키(Ctrl/Alt/Win 조합)가 조합을 확정하지 못하던 문제**: 조합 중 Ctrl+S를
  누르면 마지막 음절이 빠진 채 저장됐다. 이제 조합 중 단축키는 **음절을 먼저 확정하고 키는
  그대로 통과**시켜 앱이 원래 타이밍에 기능키를 처리한다. (1차 구현의 eat+재전달 방식은
  실기에서 주입 키가 큐의 사용자 입력에 추월당해 Ctrl 없이 자모로 오입력되는 문제(ㅊ/ㅍ)가
  발견되어 통과 방식으로 재설계.)
- **레거시(CUAS) 앱에서 조합 중 엔터를 누르면 마지막 음절이 소실되던 문제**(AkelPad
  "가나다라"+Enter → "가나다"): 경계키(엔터/방향키 등) 재전달을 ~30ms 지연해 확정 음절이
  앱에 먼저 도달하게 했다. (스페이스는 v0.11.0의 단일 삽입 유지.)
- **조합 미리보기 칩이 매 키마다 크기가 바뀌며 여러 번 깜빡이던 문제**(메모장): 사후
  자기보정이 편집세션 rect와 시스템 캐럿 rect의 항구적 높이 차이를 서로 "교정"하며 진동
  — 보정을 캐럿 위치가 실제로 움직였을 때만 발동하도록 수정.
- **설정 창 Add로 추가한 외부 `.jmt` 자판이 재시작 후 사라지던 문제**: 추가 시
  `%APPDATA%\Jamotong\layouts`(사용자 자판 저장소)로 복사하고, 시작 시 DLL 옆 폴더와 함께
  자동 로드한다.
- 설정 창에서 마지막 켜진 자판을 삭제해 "켜진 자판 0개" 상태를 만들 수 있던 문제(적용 시
  자동 복구 포함), Add 후 적용 전 삭제한 자판의 메모리 누수.
- IME 비활성화 시 열려 있던 한자 후보창이 정리되지 않던 문제(해제된 서비스 참조 위험).
- 잘못된 `.jmt`(범위 밖 자모 인덱스, 정의 안 된 조합 글쇠, 미지의 key/mod 이름)가 "로드
  성공 후 입력만 깨지는" 대신 로드 시점에 거부된다.
- `regsvr32` 등록 시 TSF 프로파일/카테고리 등록 실패가 성공으로 숨겨지던 문제 — 실패
  코드가 설치 스크립트까지 전파된다.
- 편집 세션 내부 실패(삽입 실패 등)가 호출자에 전파되지 않던 문제.

### Changed (RFC-0004)
- 한자/훈음 사전을 IME 활성화 시가 아니라 **첫 한자 사용 시점에 로드**(모든 텍스트 앱
  프로세스에서의 불필요한 파일 IO 제거).
- 미사용 소스 `layout_parser.c/.h` 제거.

### Fixed
- 글자 크기가 섞인 문서에서 조합 미리보기 칩의 크기·위치가 한 키 늦게 따라오던 문제 —
  표시 직후 40/80/120ms에 시스템 캐럿을 재표본해 실제 위치·줄높이와 다르면 즉시
  재배치/재크기하는 사후 자기보정 추가 (Auto 크기일 때; 고정 크기는 위치만 보정).

## [0.11.0] - 2026-07-08

### Added
- **트레이 입력 표시기에 현재 자판 아이콘**(`GUID_LBI_INPUTMODE`): 자모통 활성 시 브랜딩
  아이콘 왼쪽에 자판 축약명이 표시되고 전환 시 즉시 갱신. 좌클릭=자판 순환,
  우클릭=컨텍스트 메뉴(Settings/Next layout/About).
- **선택(블록) 텍스트 한자 변환**: 텍스트를 선택하고 한자키 → 후보창 → 선택 영역 교체.
  선택 교체는 삽입 경로라 레거시(CUAS) 앱에서도 동작(단어 단위 변환의 전면 지원).

### Changed
- 프로그램 내부 메시지·설치 스크립트·배포 문서를 영어 전용으로 통일(제작자 rubidus-api).
  README와 기술 매뉴얼은 영어(기본)/한국어 파일로 분리.
- 조합 중 스페이스: 확정 음절+공백을 **한 번의 삽입**으로 처리(합성 키 재전달 제거).

### Fixed
- 레거시(CUAS) 앱에서 어절 마지막 음절이 간헐적으로 소실되던 문제 — 합성 경계키와
  결과 문자 전달의 경합이 원인(스페이스 경계의 단일 삽입화로 해결).
- 미리보기 칩이 레거시 앱에서 직전 글자 위에 겹치거나 한 박자 늦게 따라오던 문제 —
  낡은 좌표 탐지(누적 보정)로 해결. 네이티브 앱은 영향 없음.

## [0.10.0] - 2026-07-07

### Added
- 한자 후보창에 **훈음(뜻·음) 표시** — `1. 家  집 가` (미수록 한자는 `U+XXXX` 폴백).
- 후보창 탐색 강화: ↑↓ 선택 하이라이트(페이지 경계 자동 넘김), Enter 선택,
  ←→/PgUp/PgDn/Space 페이지, **마우스 클릭 선택**, **우상단 ✕(닫기) 버튼**.
- **유니코드 코드포인트 직접 입력**: `Ctrl+Alt+U` → 16진수 입력 팝업(실시간 미리보기) → Enter 삽입.
- **시스템 트레이 모니터링 툴**(jamotong.exe): 현재 자판 축약명을 트레이 아이콘으로 표시
  (자모통 활성=파랑/비활성=회색, 1.5초 갱신), 좌클릭=설정, 우클릭 메뉴(설정/입력 테스트/정보/종료).
- 조합 미리보기 오버레이 **글꼴 크기 Auto/고정(px) 옵션** — 캐럿 크기를 작게 보고하는
  일부 앱(터미널류) 대응.
- 버전 규약 도입: `src/version.h`에 MAJOR/MINOR/PATCH 상수 일원화(리소스·정보창 파생).

### Changed
- 설치/삭제 스크립트 정리: 무음 등록(`regsvr32 /s`), 실패 시 스크립트 내 오류 안내 후 중단,
  폐기된 IMM32(.ime) 설치 단계 제거(삭제 스크립트는 구버전 잔재를 조용히 정리).
- 포커스/문서 이동 시 후보창을 콜백 경유로 취소해 참조 정리(잔류·멈춤 방지).

### Removed
- 사용처가 없어진 "Legacy app support" 옵션(설정창·설정 파일) 제거 — 커밋 전용 엔진은
  앱 종류와 무관하게 동일 동작. (구버전 설정 파일의 해당 줄은 무시됨)

## [0.9.0] - 2026-07-07

### Added
- **조합 미리보기 플로팅 오버레이**: 조합 중 음절을 캐럿 위치의 반투명 칩으로 표시

### Fixed
- 전면 코드 감사: 설정 로드의 리소스 누수/빈 자판 문제(병합 로드로 재설계),
  키 예측 경로의 락 누락, `TF_IAS_NOQUERY` NULL 역참조 위험,
  설정 스레드 핸들 누수, 모달 루프의 WM_QUIT 유실, 윈도 클래스 소유/해제
  (DLL 재로드 크래시 방지), wide-scanf 변환 지정자 표준화(%l 계열) 등.
