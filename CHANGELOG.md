# Changelog

이 프로젝트의 주요 변경 사항을 기록한다. [Keep a Changelog](https://keepachangelog.com) 형식과
[Semantic Versioning](https://semver.org)(MAJOR.MINOR.PATCH — 비호환 변경 / 신규 기능 / 버그 수정)을 따른다.

## [Unreleased]

## [0.11.1] - 2026-07-08

### Changed
- 기술 매뉴얼(한/영) 대폭 갱신: §12 "커밋 전용 이후의 실전 교훈" 신설(오버레이·CUAS 낡은 좌표·
  마지막 음절 소실·트레이 모드 아이콘·선택 한자 변환·진단 방법론 — 각 문제/원인/최종 해법),
  §10 함정 4건 추가(wide-scanf %l, 윈도 클래스 소유권, TF_IAS_NOQUERY, 락 범위).

## [0.11.0] - 2026-07-08

### Added
- **트레이 입력 표시기에 현재 자판 아이콘**(`GUID_LBI_INPUTMODE`): 자모통 활성 시 브랜딩
  아이콘 왼쪽에 자판 축약명이 표시되고 전환 시 즉시 갱신. 좌클릭=자판 순환,
  우클릭=컨텍스트 메뉴(Settings/Next layout/About).
- **선택(블록) 텍스트 한자 변환**: 텍스트를 선택하고 한자키 → 후보창 → 선택 영역 교체.
  선택 교체는 삽입 경로라 레거시(CUAS) 앱에서도 동작(단어 단위 변환의 전면 지원).

### Changed
- 프로그램 내부 메시지·설치 스크립트·배포 문서를 영어 전용으로 통일(제작자 rubidus-api).
  README와 기술 매뉴얼은 영어(기본)/한국어 파일로 분리.
- 조합 중 스페이스: 확정 음절+공백을 **한 번의 삽입**으로 처리(합성 키 재전달 제거).

### Fixed
- 레거시(CUAS) 앱에서 어절 마지막 음절이 간헐적으로 소실되던 문제 — 합성 경계키와
  결과 문자 전달의 경합이 원인(스페이스 경계의 단일 삽입화로 해결).
- 미리보기 칩이 레거시 앱에서 직전 글자 위에 겹치거나 한 박자 늦게 따라오던 문제 —
  낡은 좌표 탐지(누적 보정)로 해결. 네이티브 앱은 영향 없음.

## [0.10.0] - 2026-07-07

### Added
- 한자 후보창에 **훈음(뜻·음) 표시** — `1. 家  집 가` (미수록 한자는 `U+XXXX` 폴백).
- 후보창 탐색 강화: ↑↓ 선택 하이라이트(페이지 경계 자동 넘김), Enter 선택,
  ←→/PgUp/PgDn/Space 페이지, **마우스 클릭 선택**, **우상단 ✕(닫기) 버튼**.
- **유니코드 코드포인트 직접 입력**: `Ctrl+Alt+U` → 16진수 입력 팝업(실시간 미리보기) → Enter 삽입.
- **시스템 트레이 모니터링 툴**(jamotong.exe): 현재 자판 축약명을 트레이 아이콘으로 표시
  (자모통 활성=파랑/비활성=회색, 1.5초 갱신), 좌클릭=설정, 우클릭 메뉴(설정/입력 테스트/정보/종료).
- 조합 미리보기 오버레이 **글꼴 크기 Auto/고정(px) 옵션** — 캐럿 크기를 작게 보고하는
  일부 앱(터미널류) 대응.
- 버전 규약 도입: `src/version.h`에 MAJOR/MINOR/PATCH 상수 일원화(리소스·정보창 파생).

### Changed
- 설치/삭제 스크립트 정리: 무음 등록(`regsvr32 /s`), 실패 시 스크립트 내 오류 안내 후 중단,
  폐기된 IMM32(.ime) 설치 단계 제거(삭제 스크립트는 구버전 잔재를 조용히 정리).
- 포커스/문서 이동 시 후보창을 콜백 경유로 취소해 참조 정리(잔류·멈춤 방지).

### Removed
- 사용처가 없어진 "Legacy app support" 옵션(설정창·설정 파일) 제거 — 커밋 전용 엔진은
  앱 종류와 무관하게 동일 동작. (구버전 설정 파일의 해당 줄은 무시됨)

## [0.9.0] - 2026-07-07

### Added
- **조합 미리보기 플로팅 오버레이**: 조합 중 음절을 캐럿 위치의 반투명 칩으로 표시
  (문서 무접촉 — 호환성 제약 없음). 글꼴 face 사용자 설정, 설정 파일 저장/Import/Export 포함.
- **한자 데이터 구축**: Unicode® Unihan Database(kHangul) 파생 독음 테이블
  486개 음·8,900+자(교육용 기초 한자 우선 정렬, 다음자 수록) + 자체 큐레이션
  상용 한자어 331개 + 훈음 표 1,784자. 출처·라이선스 명세와 Unicode License v3 사본 동봉.
- 재현 가능한 데이터 생성기 및 데이터 무결성 검사.

### Fixed
- 전면 코드 감사: 설정 로드의 리소스 누수/빈 자판 문제(병합 로드로 재설계),
  키 예측 경로의 락 누락, `TF_IAS_NOQUERY` NULL 역참조 위험,
  설정 스레드 핸들 누수, 모달 루프의 WM_QUIT 유실, 윈도 클래스 소유/해제
  (DLL 재로드 크래시 방지), wide-scanf 변환 지정자 표준화(%l 계열) 등.

## [0.8.0] - 2026-07-06

### Changed
- **입력 엔진을 "커밋 전용"으로 확정**: 확정 음절만 문서에 삽입. 레거시(IMM32 브리지)
  앱을 포함한 모든 클래식 앱에서 동일 동작 — 앱 감지·첫 입력 파괴 없음.
  (배경과 근거는 `winapi-c-ime-manual.md` §8 참조)
- 한자 변환을 커밋 전용 모델에 맞게 삽입 방식으로 정리.

### Added
- 종합 기술 매뉴얼 `winapi-c-ime-manual.md`: WinAPI+C만으로 IME를 구현하는 전 과정
  (COM 기초 → TSF → 시행착오와 결론).

### Removed
- 실효성이 없어진 조합/감지/우회 경로들(코드 정리).

## [0.7.0] - 2026-07-06

### Added
- IMM32 IME(.ime) 실험 구현 — 조합/확정/한영/한자까지 빌드 완료.

### Deprecated
- 실기 확인 결과 **Windows 11이 서드파티 IMM32 IME 활성화를 차단**(목록 표시는 되나
  비활성)함을 확인, IMM32 경로 폐기. 결론은 0.8.0의 커밋 전용 TSF로 승계.

## [0.6.0] - 2026-07-06

### Added
- Windows 11 입력 표시기 브랜딩 아이콘("자모통") — 모던 프로파일 등록 방식 적용.
- 설정창 단축키 `Ctrl+Alt+K`, 설정 영속화(`%APPDATA%\Jamotong\config.ini`),
  독립 설정/자판 편집 앱(jamotong.exe).

## [0.5.0] - 2026-07-05

### Fixed
- 조합 표시 문제를 표준 구현(MS TSF SampleIME, MIT) 대조로 교정 — SetText 플래그,
  조합 시작 시퀀스, 캐럿 처리. 메모리/COM 프로토콜 1차 감사.

## [0.1.0 ~ 0.4.0] - 2026-06

### Added
- TSF 텍스트 서비스 골격(순수 C23: COM 클래스 팩토리·등록·키 이벤트 싱크).
- 한글 오토마타(2벌식/세벌식(안)) — 조합·도깨비불·백스페이스(자소 단위).
- 자판 시스템: 내장 자판 + `.jmt` 사용자 자판(정적 맵/한글 조합/코드 자판), 자판 순환 단축키.
- 한자·특수문자 후보창, 모아치기(동시치기), 언어바 아이콘, 설정창(자판/단축키/옵션 탭).
- 네이티브 테스트(오토마타/설정, ASan/UBSan 게이트)와 MinGW 크로스 빌드 체계.
